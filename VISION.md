# Bee Swarm 架构愿景 —— 就当是聊天

> 这不是 PRD，不是 spec，是思路的锚点。随时改、随时补。

---

## 1. 我要做什么？

一个面向 **学术研究与写作创作** 的 **工作管理系统**。

不是 "搞个 agent 框架玩"，而是真的要做成一套能管理「思考 → 执行 → 验证 → 沉淀」全链路的系统。

---

## 2. 四大技术锚点

| 锚点 | 来源 | 在我这里变成什么 |
|------|------|-----------------|
| **nanobot tool calling** | nanobot-ai | Bee 的执行层 —— Worker Bee 的 tool-use 核心 |
| **Hermes Agent 记忆与知识管理** | hermes-agent | Bee 的认知层 —— 跨 session 的长期记忆、skill 体系、知识沉淀 |
| **Grok Build 透明化** | xAI Grok | 可观测层 —— 每一步 agent 思考、tool call、文件改动，全部可视化 |
| **Code Review Graph** | code-review-graph | 协作层 —— 变更追踪、审查链路、知识图谱化 |

这四个东西拼起来，就是 **Bee 的基底（substrate）**。

---

## 3. 当前基底：Bee Swarm 是什么？

File-as-Bus 的多 agent 协作框架。三层蜂群：

- **PM Bee** —— 把目标翻译成可执行计划
- **Centurion Bee** —— 拆解、分发、监控、聚合
- **Worker Bee** —— 实际干活（读/写/搜索/调用 tool）

记忆层已经搭了 nanobot 的 MemoryStore + GitStore，历史可回滚、可压缩（dream）。

这是 **Phase 1** 的交付物，能跑通，但只是骨架。

---

## 4. Phase 2+ 要堆什么？

### 4.1 Skills & Features（大量）

Hermes 的 skill 体系要全盘接入。每一个 skill 都是一只蜂能调用的能力：

- `research` —— 文献检索、arxiv 搜索、知识提取
- `writing` —— 大纲生成、段落润色、引用管理
- `review` —— 自我审查、交叉验证、逻辑漏洞扫描
- `coding` —— 代码执行、数据分析、可视化
- `planning` —— 任务拆解、依赖分析、进度追踪

不是预置几个函数，而是 **skill 即插件**，热插拔、可版本化、可共享。

### 4.2 Dashboard（全流程可视化）

不只是看 task pool 里几个 pending。要的是：

- **管理全闭环** —— 从「产生一个想法」到「交付一篇论文/一个章节」，每一步状态可见
- **知识全闭环** —— 所有中间产物（笔记、草稿、引用、 rejected 的思路）都进知识库，不是丢进垃圾桶
- **WorldBee 数字孪生** —— 用户的知识世界在系统里持续积累、不断进化

Dashboard 不是 BI 报表，是 **创作者的操作系统界面**。

### 4.3 WorldBee：数字孪生与世界知识

这是整个系统的终极形态：

- 用户的全部工作产出（论文、笔记、代码、数据）构成一个 **活的知识图谱**
- 不是静态存档，是 **持续被 agent 读取、引用、扩展、修正** 的有机体
- 新任务自动关联已有知识，避免重复造轮子
- 长期演进：用得越久，WorldBee 越懂用户的研究方向和思维习惯

## 5. 目标用户画像

- 学术研究者（读 paper、写论文、做综述）
- 内容创作者（深度写作、专栏、书籍）
- 知识工作者（需要大量阅读 → 思考 → 输出闭环的人）

不是给 casual user 的聊天工具，是 **严肃工作系统**。

---

## 6. 非目标（现在就写下来，防止 scope creep）

- ❌ 通用聊天机器人
- ❌ 社交/协作编辑（先单人闭环，多人以后再说）
- ❌ 实时多媒体生成（先文本/代码/数据，图像音频后续）
- ❌ 替代 Word / Overleaf（是增强层，不是替换层）

---

## 7. 关键设计原则

1. **File-as-Bus 不动摇** —— 零 infra、零消息队列、可观测、可调试
2. **Git 即时间轴** —— 一切可回滚、一切有历史、一切可审计
3. **Skill 即 API** —— 新增能力不改动核心，只加 skill
4. **透明即信任** —— agent 每一步思考必须可见，黑箱不可接受
5. **渐进增强** —— 先能用，再好用，再离不开
